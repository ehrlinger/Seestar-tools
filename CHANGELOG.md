## [Unreleased]

### Added
- **`inapp_inventory.py`** — catalogs the Seestar's in-app `Stacked_*` previews (the on-board stacks in each `<target>/` folder, distinct from the `*_sub/` raw subs and Siril stacks). Reads everything from the self-describing filename (`Stacked_<subs>_<target>_<exp>s_<filter>_<datetime>`) — no FITS headers — and prints a caption-ready markdown table (or `--csv`) of the best (most-subs) stack per target, with integration time and `.fit`/`.jpg` presence. `--all` lists every stack version; `--write` saves `Astrophotography/In-App Stacks Inventory.md` to the vault (via `SEESTAR_VAULT_INV`, same as `count_subs`) plus a copy at the archive root. `sync_seestar.sh` runs it as **Step 4b** so the in-app catalog refreshes on every sync. Covered by unit tests (`parse_stacked_name`, `find_inapp_stacks`, `best_per_target`, `resolve_inapp_inventory_path`).
- **Safe destination control in `sync_seestar.sh`.** New `--src` / `--dst` (alias `--dest`) command-line overrides take precedence over `seestar.conf`. A real (non-dry-run) sync now prints the resolved `SRC → DST` and asks for confirmation before touching anything; `--yes` / `-y` skips the prompt for scripted runs, and an unattended run without `--yes` refuses rather than guessing. This closes the "wrong-folder sync" trap where a stale/duplicate config silently pointed at the wrong NAS path.
- **Exposure-sort wired into the pipeline.** `sync_seestar.sh` gained **Step 3b**: after organising into `lights/`, it runs `sort_by_exptime.py --all` to put each target into its canonical, Siril-ready shape on every sync — grouped by **mount mode**, so a both-modes target splits into `<target>_sub/altaz/lights/` + `eq/lights/` (alt-az and EQ stacked separately) and a single-mode target stays flat at `<target>_sub/lights/`.
- **`seestar.conf.example`** — the config template the README and `.gitignore` referenced but that was missing from the repo. Copy it to the gitignored `seestar.conf`.
- **Config-driven inventory path.** `count_subs.py` gained `--inventory PATH`; the inventory location resolves `--inventory` → `SEESTAR_VAULT_INV` (exported from `seestar.conf` by `sync_seestar.sh`) → built-in default, instead of being hardcoded.
- **`purge_siril_cruft.py`** — reclaims archive space by deleting three disposable categories from `*_sub/` folders: Siril `process/` working dirs (regenerated on every re-stack), `Copy #N of …` Finder duplicates, and double-processed `starless_starless_*` / `starmask_starless_*` files. Dry-run by default (`--apply` to delete); idempotent; NAS keepalive. A `Copy #N of X` is removed only when its original `X` is present in the same folder, so orphaned copies are never the sole casualty. After deletion it hands surviving `lights/` folders to `sort_by_exptime.py`. First archive-wide dry-run reported ~769 GB / ~53,400 files reclaimable, ~99.99% of it `process/` scratch.
- **Cross-platform support.** Linux and Windows install/path notes throughout the README. Recommended Windows workflow is WSL2; native PowerShell users can invoke the Python scripts directly, only `sync_seestar.sh` is bash-bound.
- **Platform-aware Siril script discovery** in `batch_stack.py` — `SCRIPT_SEARCH_PATHS` now builds per-platform (macOS, Linux including flatpak, Windows incl. `%APPDATA%` and Program Files).
- **`seestar.conf` loading in `sync_seestar.sh`.** The script's comment previously claimed it loaded the conf but the code didn't actually source it — only the macOS `/Volumes` defaults were used. Now it sources `seestar.conf` if present, then falls back to platform-aware defaults via `uname -s`.
- **`seestar_common.py`** — a small dependency-free shared module (currently the `_trash`/`scripts` directory-exclusion helper `is_in_excluded()`), importable by every standalone script.
- **Unit test suite** (`tests/test_parsing.py`, 112 tests using stdlib `unittest` — no install required). Covers `canonical_target_name`, `is_processed`, `safe_dest`, `group_by_target`, `_EXPTIME_RE`, `format_duration`, `normalize`, `canonical_folder_name`, `deduplicate`, `new_name`, `merge_into_existing`, `STACKED_RE`, plus `sort_by_exptime.exptime_label` / `EXPTIME_DIR_RE` / `is_already_sorted` / `find_target_dirs` / `mount_mode` / `plan_layout` / `normalize_target`, `organize_subs.clean_sub_folder` (lazy-`lights/` scaffold), `count_subs.resolve_inventory_path`, `batch_stack.find_stack_units`, `purge_siril_cruft.find_target_dirs`, and `seestar_common.is_in_excluded` / `_trash` exclusion across all discovery scanners.
- **GitHub Actions CI** (`.github/workflows/tests.yml`): matrix of macOS / Ubuntu / Windows × Python 3.10 / 3.12 / 3.13, plus a shellcheck job for `sync_seestar.sh`.
- **PR template** (`.github/pull_request_template.md`) and a Contributing / branch-strategy section in the README documenting topic-branch + squash-merge workflow and the recommended branch-protection toggles.
- **LICENSE** (MIT).
- **CHANGELOG.md** (this file).

### Fixed
- **The pipeline no longer treats anything under `_trash/` (or `scripts/`) as a target.** Target discovery used a recursive `rglob`, so `rename`, `organize`, `count`, and `batch_stack` would pick up folders inside a `_trash/` directory in the archive and process whatever was set aside there — e.g. reporting "would rename 4 folder(s)" for empty stray folders living under `_trash/strays/`, and (if real subs were ever trashed) renaming/sorting/counting them. Recursive scanners now **filter out** any path under an excluded directory name (`_trash`, `scripts`, case-insensitively) via a new shared `seestar_common.is_in_excluded()` helper, and `sort_by_exptime.py`'s redesigned (non-recursive) `find_target_dirs()` enforces the same exclusion. Covered by unit tests across every discovery scanner.
- **`batch_stack.py` discovers stack units regardless of layout.** Discovery, "has lights", and stale-stack detection previously assumed a flat `<target>_sub/lights/`, so after Step 3b moved subs into per-group subfolders, batch stacking silently skipped every grouped target as `[no lights]`. Discovery now finds **stack units** (any directory holding a `lights/` of raw FITS), which matches every layout — a flat `<target>_sub/lights/`, a per-mode `altaz/`/`eq/` folder, or a legacy `<exp>s/` folder; each is its own unit so different mount modes stack separately. Works whether pointed at the archive root, a `<target>_sub`, or an individual sub-folder. `lights/` counting now skips macOS `._` resource-fork files so sub counts (and staleness comparisons) aren't inflated. A regression test pins the case where an empty top-level `<target>_sub/lights/` scaffold sits beside a populated grouped `lights/`: only the populated unit is stacked, never the empty root — so Siril's `cd lights` always lands on real subs.
- **Subs are now grouped by mount mode, not by raw exposure.** `sort_by_exptime.py` previously nested subs into one folder *per exact exposure* (`10s/`, `20s/`, `30s/` …). That was wrong on two counts: (1) for a single-exposure target like a 634×20 s M 13 it nested for no reason and fought a hand-run of Siril (`Seestar_Preprocessing.ssf` opens with `cd lights`, but the subs were a level down and the top-level `lights/` was an empty scaffold, so a GUI run on `M_13_sub` stacked nothing); and (2) it **over-split** EQ targets — M 57's 20 s and 30 s subs (both EQ) were stacked as two separate half-integrations instead of one 1,488-sub stack. The pipeline now groups by **mount mode** — a Seestar shoots ~10 s in alt-az and 20 s+ in EQ, and those must stack separately (field rotation), while different lengths within a mode belong in one deeper stack. So a single-mode target is **flat** (`<target>_sub/lights/`) and a both-modes target is **split** (`<target>_sub/altaz/lights/` + `eq/lights/`), combining 20 s+30 s EQ into the `eq/` stack. `normalize_target()` **converges** each target to this shape from any starting state — flat, mode-split, or the legacy per-exposure layout — idempotently; existing per-exposure targets are migrated on the next sort (e.g. M 57 → flat `eq`, M 63's 10s/20s/30s → `altaz` + combined `eq`). Mode is inferred from exposure (`< ~15 s` ⇒ alt-az, else EQ; see `MODE_BOUNDARY_SECONDS`). New `mount_mode()` + pure `plan_layout()` planner + `normalize_target()` driver, covered by 18 unit tests; dry-run output reports exactly what a real run moves and removes. Relatedly, `organize_subs.py` no longer pre-creates an empty `lights/` scaffold — it makes `lights/` lazily, only when there's a raw FITS to file into it, so it can't re-introduce the misleading empty folder over an already-grouped target.
- **`_subs` folders are no longer skipped by the exposure-sort and purge tools.** `sort_by_exptime.py --all` and `purge_siril_cruft.py`'s default target discovery matched only names ending in `_sub`, silently ignoring `_subs` targets (every other tool handles both). Both now match `_sub` and `_subs` via a shared `find_target_dirs()` helper, so a `NGC 6946_subs` target is exposure-sorted on sync and purged like any other.
- **Incremental syncs no longer strand new subs in space-named folders.** `rename_seestar_folders.py` previously *skipped* a rename when the canonical destination already existed (`M 51_sub` → `M_51_sub` where `M_51_sub` is already present), leaving the freshly-synced subs in the space-named folder; the later organize/sort steps never consolidated them, so each incremental sync grew a parallel `M 51_sub` / `M 57_sub` / `NGC 6946_sub` instead of merging into the canonical `<target>_sub/<exptime>s/lights/` tree. The rename step now **merges** the donor folder into the existing destination: every file is moved into the matching relative location under the destination (raw root subs land at the destination root, where Step 3/3b file them by exposure), deduped by filename **and size** against everything already in the destination so re-delivered subs don't double-count, and the emptied donor folder is removed. A same-name file with a different size is never treated as a duplicate — it is preserved under a collision-safe `_dupN` name rather than unlinked, so no real data can be silently dropped. Dry-run-by-default and idempotent; the already-sorted `<exptime>s/lights/` subs are never touched. New `merge_into_existing()` helper, covered by 6 unit tests.
- **`canonical_target_name` no longer strips target catalog numbers.** The previous `_SESSION_SUFFIX_RE` had a bare `\d+` branch that matched the target's own catalog number once `_sub` was stripped, so `M 51_sub` returned `"M"` instead of `"M 51"`, collapsing every `M N` and `NGC N` folder into just its prefix letter. This broke the `--merge` feature in `organize_subs.py` (then named `cleanup_seestar.py`), which would incorrectly group multi-target archives together. Fix: removed the bare-digit branch, required a separator before remaining session markers, and added explicit handling for middle `_sub_N` / `_subs_N` suffixes so `M 51_sub_2 → M 51` works.

  **Impact:** if you had previously run `--merge` on an archive with multiple Messier or NGC targets, those merges may have moved subs into the wrong target folder. Recommend reviewing archives before re-running.
- **CSV writer encoding** in `count_subs.py` now explicitly sets `encoding="utf-8"`. Previously it defaulted to the platform locale (`cp1252` on Windows), which would mangle non-ASCII target names.

### Changed
- **Sync now keeps the Seestar in-app stack JPGs.** `sync_seestar.sh` Step 1 previously excluded *all* `*.jpg`. It now excludes only the per-sub thumbnails under `*_sub/` (thousands of `Light_*.jpg` / `*_thn.jpg`) and **keeps the in-app `Stacked_*.jpg` previews** in the `<target>/` folders — the not-reproducible-in-Siril renderings whose filenames encode provenance (`Stacked_<N>_<target>_<exp>s_…` → sub count + exposure).
- **Renamed two scripts for clarity** (they read as two "clean*" tools doing opposite things):
  `cleanup_seestar.py` → **`organize_subs.py`** (it *organises* subs into `lights/` and merges nights — it never deleted data), and
  `clean_seestar_archive.py` → **`purge_siril_cruft.py`** (it *purges* regenerable Siril `process/` scratch + duplicates).
- README restructured for cross-platform: per-OS config examples, per-OS Siril and Python install notes, rsync-on-Windows guidance, new Windows workflow section, Tests section, Contributing section.

## [0.1.0] — 2026-05-24

### Added
- Initial commit: full Seestar S50 sync pipeline.
- `sync_seestar.sh` — orchestrates rsync → rename → organise → count + inventory.
- `rename_seestar_folders.py` — removes spaces from `_sub`/`_subs` folder names.
- `cleanup_seestar.py` — moves raw FITS into `lights/`, merges multi-night sessions, removes empty folders.
- `count_subs.py` — counts subs per target, updates Obsidian inventory markdown.
- `batch_stack.py` — batch-runs Siril's `Seestar_Preprocessing` script across pending folders.
- `seestar.conf.example` config template, gitignored personal `seestar.conf`.
